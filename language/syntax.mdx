Ruby's syntax is expressive and readable. This page covers the core building blocks: variables, literals, control flow, and method definitions.

## Variables

Ruby has five kinds of variables, distinguished by their prefix.

<Tabs>
  <Tab title="Local">
    Local variables are lowercase (or start with `_`) and are scoped to the enclosing method, block, or file.

    ```ruby
    name = "Alice"
    age  = 30
    _tmp = "temporary"
    ```
  </Tab>
  <Tab title="Instance (@)">
    Instance variables belong to a specific object and are accessible in any method of that object.

    ```ruby
    class Person
      def initialize(name)
        @name = name   # instance variable
      end

      def greet
        "Hello, #{@name}!"
      end
    end
    ```
  </Tab>
  <Tab title="Class (@@)">
    Class variables are shared across a class and all of its subclasses.

    ```ruby
    class Counter
      @@count = 0

      def initialize
        @@count += 1
      end

      def self.count
        @@count
      end
    end

    Counter.new
    Counter.new
    Counter.count # => 2
    ```
  </Tab>
  <Tab title="Global ($)">
    Global variables are accessible anywhere in the program. Their use is generally discouraged.

    ```ruby
    $global_setting = "enabled"

    def check
      puts $global_setting
    end

    check # => "enabled"
    ```
  </Tab>
  <Tab title="Constants">
    Constants begin with an uppercase letter. By convention, all-caps names are used for true constants.

    ```ruby
    PI      = 3.14159
    MAX_SIZE = 100
    AppName  = "MyApp"  # also valid — title-case for class/module names
    ```

    <Warning>
      Ruby will warn if you reassign a constant, but it does not raise an error.
    </Warning>
  </Tab>
</Tabs>

## String Literals

Ruby offers several string syntaxes, each with different interpolation and escape behavior.

<Tabs>
  <Tab title="Double-quoted">
    Double-quoted strings support interpolation with `#{}` and escape sequences like `\n`, `\t`.

    ```ruby
    name = "world"
    greeting = "Hello, #{name}!"   # => "Hello, world!"
    multiline = "Line one\nLine two"
    ```
  </Tab>
  <Tab title="Single-quoted">
    Single-quoted strings are literal — no interpolation or most escape sequences (only `\\` and `\'` are processed).

    ```ruby
    path = 'C:\Users\name'     # backslashes kept as-is
    literal = '#{not interpolated}'
    ```
  </Tab>
  <Tab title="Heredoc">
    Heredocs allow multi-line strings. Use `<<~` to strip leading whitespace.

    ```ruby
    message = <<~TEXT
      Dear #{name},
      Your account is ready.
      Regards, Support
    TEXT

    # Squiggly heredoc strips common leading indentation
    sql = <<~SQL
      SELECT *
      FROM users
      WHERE active = true
    SQL
    ```
  </Tab>
  <Tab title="%q / %Q">
    `%q` is equivalent to single-quoting; `%Q` (or `%`) to double-quoting.

    ```ruby
    str1 = %q(single 'quoted' string)
    str2 = %Q(double "quoted" with #{1 + 1} interpolation)
    str3 = %(shorthand for %Q)
    ```
  </Tab>
</Tabs>

## Symbols

Symbols are immutable, interned identifiers. They are memory-efficient alternatives to strings when identity matters more than content.

```ruby
status = :active
key    = :user_name

# Symbols with spaces use quoted syntax
fancy  = :"first name"

# Symbols are commonly used as hash keys
options = { color: :red, size: :large }

# Symbol to proc shorthand
words = ["hello", "world"]
words.map(&:upcase) # => ["HELLO", "WORLD"]

# Comparison — all occurrences of the same symbol are identical
:foo.object_id == :foo.object_id # => true
```

## Numbers

Ruby supports integers, floats, rational numbers, and complex numbers. Underscores may be used as visual separators.

```ruby
# Integers
population = 8_100_000_000
hex        = 0xFF         # => 255
octal      = 0o77         # => 63
binary     = 0b1010_1010  # => 170

# Floats
pi      = 3.14159_26535
science = 1.5e10          # => 15000000000.0

# Rational — exact fractions (no floating-point error)
half    = 1/2r            # => (1/2)
third   = Rational(1, 3)  # => (1/3)
result  = half + third    # => (5/6)

# Complex numbers
c = 2 + 3i                # => (2+3i)
c.real                    # => 2
c.imaginary               # => 3
```

## Arrays and Hashes

<Tabs>
  <Tab title="Arrays">
    ```ruby
    # Literal array
    fruits = ["apple", "banana", "cherry"]

    # Word array shorthand
    words = %w[one two three]      # => ["one", "two", "three"]
    syms  = %i[foo bar baz]        # => [:foo, :bar, :baz]

    # Access and mutation
    fruits[0]           # => "apple"
    fruits[-1]          # => "cherry"
    fruits.push("date") # => ["apple", "banana", "cherry", "date"]
    fruits << "elderberry"

    # Nested arrays
    matrix = [[1, 2], [3, 4], [5, 6]]
    matrix[1][0]        # => 3
    ```
  </Tab>
  <Tab title="Hashes">
    ```ruby
    # Rocket-syntax hash (any key type)
    legacy = { "name" => "Alice", 1 => "one" }

    # Symbol-key shorthand (most common)
    person = { name: "Alice", age: 30, active: true }

    # Access
    person[:name]       # => "Alice"
    person.fetch(:age)  # => 30
    person.fetch(:missing, "default") # => "default"

    # Iteration
    person.each { |key, value| puts "#{key}: #{value}" }

    # Transformation
    person.map { |k, v| [k, v.to_s] }.to_h
    # => { name: "Alice", age: "30", active: "true" }
    ```
  </Tab>
</Tabs>

## Ranges

Ranges represent a sequence between two values. The `..` form is inclusive; `...` excludes the end.

```ruby
inclusive = 1..10     # 1, 2, 3, ..., 10
exclusive = 1...10    # 1, 2, 3, ..., 9

# Iteration
(1..5).each { |n| print "#{n} " }  # => 1 2 3 4 5

# Membership
(1..100).include?(42)   # => true
("a".."z").include?("m") # => true

# Slicing arrays
arr = [10, 20, 30, 40, 50]
arr[1..3]               # => [20, 30, 40]
arr[1...3]              # => [20, 30]

# Endless and beginless ranges (Ruby 2.6+)
arr[2..]                # => [30, 40, 50]
arr[..1]                # => [10, 20]
```

## Control Flow

<Tabs>
  <Tab title="if / unless">
    ```ruby
    # Standard if
    if age >= 18
      puts "adult"
    elsif age >= 13
      puts "teenager"
    else
      puts "child"
    end

    # unless is the opposite of if
    unless logged_in
      redirect_to :login
    end

    # Inline (modifier) form
    puts "adult" if age >= 18
    raise "Required!" if value.nil?
    ```
  </Tab>
  <Tab title="while / until">
    ```ruby
    # while — execute while condition is true
    i = 0
    while i < 5
      print "#{i} "
      i += 1
    end
    # => 0 1 2 3 4

    # until — execute until condition is true
    count = 10
    until count.zero?
      count -= 1
    end

    # loop with break
    loop do
      input = gets.chomp
      break if input == "quit"
      puts "You said: #{input}"
    end
    ```
  </Tab>
  <Tab title="case / when">
    ```ruby
    # Simple case
    grade = "B"
    result = case grade
    when "A"       then "Excellent"
    when "B", "C"  then "Good"
    when "D"       then "Passing"
    else                "Failing"
    end

    # Pattern matching with ranges
    case score
    when 90..100 then "A"
    when 80...90 then "B"
    when 70...80 then "C"
    else              "F"
    end

    # Type checking with case
    case value
    when String  then puts "It's a string"
    when Integer then puts "It's an integer"
    when Array   then puts "It's an array"
    end
    ```
  </Tab>
</Tabs>

## Blocks, Procs, and Lambdas

Ruby has three types of callable objects. Blocks are the most common — they appear after method calls using `do...end` or `{...}`.

<Tabs>
  <Tab title="Blocks">
    ```ruby
    # do...end block (conventional for multi-line)
    [1, 2, 3].each do |n|
      puts n * 2
    end

    # Curly-brace block (conventional for single-line)
    [1, 2, 3].map { |n| n ** 2 }  # => [1, 4, 9]

    # Capturing a block with &block parameter
    def run_twice(&block)
      block.call
      block.call
    end

    run_twice { puts "Hello!" }
    ```
  </Tab>
  <Tab title="Procs">
    ```ruby
    # Proc — a saved block
    doubler = Proc.new { |n| n * 2 }
    doubler.call(5)   # => 10
    doubler.(5)       # => 10 (shorthand)
    doubler[5]        # => 10 (shorthand)

    # proc{} is a shorthand
    greeter = proc { |name| "Hello, #{name}!" }
    greeter.call("Alice")  # => "Hello, Alice!"

    # Procs return from the enclosing method
    def example
      p = Proc.new { return "from proc" }
      p.call
      "after proc"   # never reached
    end
    ```
  </Tab>
  <Tab title="Lambdas">
    ```ruby
    # Lambda — like a Proc but with strict arity and local return
    square = lambda { |n| n ** 2 }
    square.call(4)   # => 16

    # Arrow syntax (Ruby 1.9+)
    add = ->(a, b) { a + b }
    add.call(3, 4)   # => 7
    add.(3, 4)       # => 7

    # Lambdas check argument count
    square.call(4, 5) # => ArgumentError: wrong number of arguments

    # Lambda return only exits the lambda, not the enclosing method
    def example
      l = lambda { return "from lambda" }
      l.call
      "after lambda"  # this IS reached
    end
    example  # => "after lambda"
    ```
  </Tab>
</Tabs>

<Note>
  Use lambdas when you need strict argument checking and local return semantics. Use procs or blocks for more relaxed callable objects passed to iterators.
</Note>

## Methods

<Steps>
  <Step title="Basic method definition">
    ```ruby
    def greet(name)
      "Hello, #{name}!"  # last expression is the return value
    end

    greet("Alice")  # => "Hello, Alice!"
    ```
  </Step>
  <Step title="Optional and default arguments">
    ```ruby
    def connect(host, port = 443, protocol = "https")
      "#{protocol}://#{host}:#{port}"
    end

    connect("example.com")            # => "https://example.com:443"
    connect("example.com", 8080)      # => "https://example.com:8080"
    connect("example.com", 80, "http") # => "http://example.com:80"
    ```
  </Step>
  <Step title="Keyword arguments">
    ```ruby
    def create_user(name:, age:, role: "member")
      { name: name, age: age, role: role }
    end

    create_user(name: "Alice", age: 30)
    # => { name: "Alice", age: 30, role: "member" }

    create_user(age: 25, name: "Bob", role: "admin")
    # => { name: "Bob", age: 25, role: "admin" }
    ```
  </Step>
  <Step title="Splat arguments">
    ```ruby
    # *args — collects extra positional arguments into an array
    def sum(*numbers)
      numbers.sum
    end

    sum(1, 2, 3, 4)  # => 10

    # **kwargs — collects extra keyword arguments into a hash
    def configure(**options)
      options.each { |k, v| puts "#{k} = #{v}" }
    end

    configure(timeout: 30, retries: 3)
    ```
  </Step>
  <Step title="Explicit return">
    ```ruby
    def find_user(id)
      return nil if id <= 0  # early return

      User.find(id)
    end
    ```
  </Step>
</Steps>
