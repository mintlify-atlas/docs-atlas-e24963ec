## Dependencies

### Required

Install these before building CRuby:

- A **C compiler** (GCC or Clang)
- [**OpenSSL**](https://www.openssl.org) 1.1.x or 3.0.x (or [LibreSSL](https://www.libressl.org)) — required for RubyGems
- [**libyaml**](https://github.com/yaml/libyaml/) 0.1.7 or later — required for RubyGems
- [**zlib**](https://www.zlib.net) — required for RubyGems

When building from the Git repository (rather than a release tarball), you also need:

- [**autoconf**](https://www.gnu.org/software/autoconf/) 2.67 or later
- [**gperf**](https://www.gnu.org/software/gperf/) 3.1 or later *(only needed if you edit source files that use gperf)*
- **Ruby** 3.1 or later *(used during the build process itself)*
- **git** 2.32 or later

### Optional

| Dependency | Purpose |
|---|---|
| [libffi](https://sourceware.org/libffi/) | Builds the `fiddle` extension |
| [gmp](https://gmplib.org) | Accelerates Bignum arithmetic |
| [rustc](https://www.rust-lang.org) 1.58.0+ | Required to build YJIT |

<Tip>
  On macOS with Homebrew, pass `--with-opt-dir` to `configure` to point at Homebrew-installed libraries:

  ```bash
  ../configure --with-opt-dir=$(brew --prefix gmp):$(brew --prefix jemalloc)
  ```

  For extension-only libraries (openssl, readline, libyaml, zlib):

  ```bash
  export CONFIGURE_ARGS=""
  for ext in openssl readline libyaml zlib; do
    CONFIGURE_ARGS="${CONFIGURE_ARGS} --with-$ext-dir=$(brew --prefix $ext)"
  done
  ```
</Tip>

## Quick Start

<Steps>
  <Step title="Get the source">
    <CodeGroup>
      ```bash From Git
      git clone https://github.com/ruby/ruby.git
      cd ruby
      ./autogen.sh
      ```

      ```bash From tarball
      tar -xzf ruby-3.0.2.tar.gz
      cd ruby-3.0.2
      ```
    </CodeGroup>
  </Step>

  <Step title="Create a build directory">
    Building in a separate directory keeps the source tree clean:

    ```bash
    mkdir build && cd build
    ```
  </Step>

  <Step title="Create the install prefix">
    ```bash
    mkdir ~/.rubies
    ```
  </Step>

  <Step title="Run configure">
    ```bash
    ../configure --prefix="${HOME}/.rubies/ruby-master"
    ```

    Add `-C` (`--config-cache`) to speed up subsequent `configure` runs.
  </Step>

  <Step title="Compile">
    ```bash
    make
    ```

    To compile in parallel, pass `-j` with the number of CPU cores:

    ```bash
    make -j$(nproc)      # Linux
    make -j$(sysctl -n hw.ncpu)  # macOS
    ```
  </Step>

  <Step title="Run the tests">
    See [Testing Ruby](/contributing/testing) to verify the build before installing.
  </Step>

  <Step title="Install">
    ```bash
    make install
    ```

    If you need `sudo` and want to avoid generating docs with different permissions:

    ```bash
    make SUDO=sudo install
    ```
  </Step>

  <Step title="Try it out">
    ```bash
    ~/.rubies/ruby-master/bin/ruby -e "puts 'Hello, World!'"
    ```
  </Step>
</Steps>

### Repository Layout After Build

```text
ruby
├── autogen.sh      # Autoconf script (step 1)
├── configure       # Generated by autogen.sh
├── build/          # Created in step 2
│   ├── GNUmakefile
│   ├── Makefile
│   └── *.o         # Compiled object files
├── include/
│   └── ruby.h      # Main public C header
├── internal/       # Internal headers used by .c files
├── lib/            # Default gems (bundler, erb, set, yaml, …)
├── spec/           # Ruby spec mirror from github.com/ruby/spec
├── test/
│   └── ruby/
├── object.c
└── *.c             # Other C source files
```

## Building with YJIT

YJIT (Yet Another JIT) compiles Ruby bytecode to native machine code at runtime. It requires Rust 1.58.0 or later.

```bash
../configure --enable-yjit --prefix="${HOME}/.rubies/ruby-master"
make
```

<Note>
  Check your Rust version with `rustc --version`. Install or update Rust via [rustup](https://rustup.rs).
</Note>

## Miniruby vs Full Ruby

Miniruby is a stripped-down build of Ruby with no external dependencies. It is built first and is required to complete the full Ruby build. You can build only Miniruby for a faster iteration cycle:

```bash
make miniruby
```

Use Miniruby when testing changes that don't involve external libraries or extensions.

## Parallel Builds

Set `MAKEFLAGS` in your shell profile to run all `make` commands in parallel by default:

```bash
# Linux (Bash/ZSH)
export MAKEFLAGS="--jobs $(nproc)"

# macOS (Bash/ZSH)
export MAKEFLAGS="--jobs $(sysctl -n hw.ncpu)"

# macOS (Fish)
export MAKEFLAGS="--jobs "(sysctl -n hw.ncpu)
```

<Warning>
  GNU make 3 has incomplete parallel execution support. Upgrade to GNU make 4 or later if you are on macOS with an older Homebrew `make`.
</Warning>

## Debugging Builds

### Debug Flags

To compile with debug symbols, disabled optimization, and verbose VM logging:

```bash
../configure cppflags="-DRUBY_DEBUG=1 -DUSE_RUBY_DEBUG_LOG=1" \
  --enable-debug-env \
  optflags="-O0 -fno-omit-frame-pointer"
```

Useful environment variables at runtime:

- `RUBY_DEBUG=gc_stress` — stress-test the garbage collector
- `RUBY_DEBUG_LOG=<file>` — write verbose VM log to a file

### Debugger Targets

Create a `test.rb` file with the Ruby script to debug, then use:

| Target | Description |
|---|---|
| `make run` | Run `test.rb` with Miniruby |
| `make runruby` | Run `test.rb` with full Ruby |
| `make lldb` | Run `test.rb` under lldb (Miniruby) |
| `make gdb` | Run `test.rb` under gdb (Miniruby) |
| `make lldb-ruby` | Run `test.rb` under lldb (full Ruby) |
| `make gdb-ruby` | Run `test.rb` under gdb (full Ruby) |

For VS Code, copy the bundled launch configurations:

```bash
cp -r misc/.vscode .vscode
```

<Note>
  If you built Ruby under `./build/`, update `.vscode/launch.json` and set `"program"` to `"${workspaceFolder}/build/ruby"`.
</Note>

### Address Sanitizer (ASAN)

ASAN detects memory safety issues in Ruby itself and in C extensions:

```bash
./autogen.sh
mkdir build && cd build
../configure CC=clang-18 \
  cflags="-fsanitize=address -fno-omit-frame-pointer -DUSE_MN_THREADS=0"
make
```

Run the test suite under ASAN (expect it to take over two hours):

```bash
RUBY_TEST_TIMEOUT_SCALE=5 SYNTAX_SUGGEST_TIMEOUT=600 make check
```

<Warning>
  ASAN requires Clang 18 or later and has only been tested on Linux. It is known **not** to work correctly on macOS due to a reverted upstream fix for multithreaded `fork`.
</Warning>

## Troubleshooting

### Unexplainable Build Errors

If you encounter build errors that don't correspond to any recent change, stale build artifacts from a previous Ruby version may be the cause. Clean everything and rebuild:

```bash
git clean -xfd
```

<Warning>
  `git clean -xfd` removes all untracked and gitignored files. Save any uncommitted work first.
</Warning>

### Code Coverage

To generate a combined C and Ruby coverage report using `gcov` and `lcov`:

```bash
./autogen.sh
../configure --enable-gcov
make
make update-coverage
rm -f test-coverage.dat
make test-all COVERAGE=true
make lcov
open lcov-out/index.html
```

Current CI coverage results are published at [rubyci.org/coverage](https://rubyci.org/coverage).
